## Overview

tinyworld is a browser-based 3D social world builder. The main entry point is `tiny-world-builder.html`, a ~1.4k-line HTML shell that loads styles from `styles/tiny-world.css` and ordered JS modules from `engine/world/00ŌĆ”99-*.js` as plain classic `<script>` tags sharing **one global scope**. Three.js r128 is self-hosted under `vendor/three/`. No bundler, no npm runtime deps.

Multiplayer is PartyKit: `party/index.js` (server, deploys with `partykit deploy`) and `47-worlds-room.js` (client). Build pipeline: `./publish.sh` copies `engine/`, `styles/`, `vendor/` ŌåÆ `dist/`; Netlify serves `dist/`. **CodeSurf auto-commits edits and main auto-pushes to GitHub ŌåÆ Netlify prod** ŌĆö branches do not guarantee isolation.

## Durable Facts

### Codebase shape
- `engine/world/*.js` ŌĆö ~40k lines across 36+ numbered modules; loaded in numeric order; every top-level name must be **unique across all modules** (duplicate `const`/`let` throws `SyntaxError` and silently kills that module's globals while others keep loading); module-private scratch globals use a prefix (e.g. `_fl`, `_sr`)
- `LandscapeEngine.js` ŌĆö superseded monolith; `getHeight`/chunk-building live in **`engine/landscape/*.js` mixins** that override it ŌĆö edit the mixins, never the monolith body
- Materials `M.*` are **shared** across meshes ŌĆö clone before mutating `.color`; never mutate in place
- `disposeGroup(group)` disposes geometries but NOT materials (materials are shared)

### Data layer contract
- `world[x][z]` = intent; `cellMeshes['x,z']` = render ŌĆö **always mutate via `setCell(x, z, opts)`**, never write `world[x][z]` directly outside init
- `setCell` updates intent, rebuilds tile/object mesh, re-renders adjacency-sensitive neighbors (fences, house clusters)

### Build & deploy
- `./publish.sh` must run after any view-facing edit or changes won't appear in prod
- `party/index.js` deploys via `partykit deploy`, NOT `publish.sh`
- `npm test` = static checks only; `npm run build` = dist generation
- Admin features (`roadmap drag`, `features admin`) silently 403 if `TINYWORLD_ADMIN_SECRET` unset ŌĆö restart `netlify dev` after setting it

### Shell / checkout traps
- `rm` is aliased interactive ŌĆö scripted `rm` silently no-ops; use `command rm -f` + verify
- cwd can drift into `~/clawd` mirror where edits auto-commit to main ŌĆö always use absolute paths
- Never `git checkout <file>` or `git reset --hard` without explicit user permission

### Style rules
- SVG glyphs only ŌĆö never reintroduce the PNG baked-icon system
- No emoji anywhere (UI, code, comments, output)
- Semicolons, 2-space indent, trailing commas, single quotes

### Voxel avatar animation states (`53-voxel-avatar.js`)

- **Adding a new state requires whitelisting it in `setState`** ŌĆö unlisted states silently collapse to idle

- States: `walk` (natural `strideCore` gait with law-of-cosines `legIK`), `jump`, `crouch`, `sit` (direct hip angles, NOT legIK), `climb`, `attack` (3-cycle sword swings), `blink`, `skydive` (visually unconfirmed), `parachute`, `rocket` (upright flight stance, built for retired skyfall)
